# Commit Convention — Code-Brain

Formato de commits:
<type>(<context>): <descrição>

## Tipos aceitos
- feat: nova funcionalidade
- fix: correção de bug
- refactor: alteração sem mudança de comportamento
- docs: documentação
- chore: manutenção
- test: testes automatizados

## Exemplo
feat(auth): implementa login via link mágico
